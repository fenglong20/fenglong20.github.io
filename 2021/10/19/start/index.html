github page + hexo + action自动部署

  • 使用github page提供的静态页面，仿照完成了hexo keep主题的博客部署，使用了github action自动化部署。reference
  • github多账号的管理 参考

准备条件

  1. 两个仓库: public 的仓库用户名.github.io和private的仓库，名称任意

  2. 本地电脑先安装node.js软件，我是通过zip解压包安装的，主要区别是需要手动配置环境变量。

    1. 在解压后的目录内新建两个目录node-global和node-cache用来保存环境。
    2. （系统or用户）环境变量PATH，将解压后的目录以及node-global目录加到PATH中。
    3. 如果一切顺利，在控制台执行node -v和npm -v有输出。
    4. 配置使用淘宝源，并安装hexo，速度飞起

  3. gitbash安装。并配置好ssh，（公钥放在github，私钥保存在本地），并申请一个有repo和workflow权限的Settings > Developer settings > Personal access tokens，这个添加到private仓库的settings > secrets > Actions secrets / New secret的name为ACTIONS_DEPLOY_KEY在自动化部署的时候有用。

    1. ~/.ssh/config文件

    2. 测试 ssh -T git@fenglong20.github.com

    3. 网速可能不稳定。可能要代理

    4. 如果之前配置过，需取消global

    5. 设置每个项目repo目录里自己的邮箱和用户名

    6. 修改仓库的地址
